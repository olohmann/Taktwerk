Editing Plist Files

Opening the Editor

Select a job in the sidebar, then click Edit in the detail pane to open the plist editor. The editor has two modes: a structured form and a raw XML editor.

Form Editor

The form editor provides fields for common launchd properties:

  • Label — The unique identifier for the job.
  • Program / Program Arguments — The executable and its arguments.
  • Run At Load — Whether the job starts automatically when loaded.
  • Keep Alive — Whether launchd should restart the job if it exits.
  • Schedule — Calendar intervals for periodic execution (minute, hour, day, weekday, month).
  • Environment Variables — Key-value pairs passed to the job.
  • Working Directory — The directory the job runs in.
  • Standard Out / Error Path — Log file paths.

Raw XML Editor

Toggle to raw XML mode to edit the plist source directly. The editor provides:

  • Live validation — A warning banner appears if the XML is malformed. The Save button is disabled until the XML is valid.
  • plutil validation — Click the Validate button to run Apple's official plutil -lint check for structural correctness beyond basic XML.

Two-Way Sync

Changes sync between the form and raw XML editors. When you switch from form to XML, the current form values are serialized to XML. When you switch back, the XML is parsed into the form fields.

Tip: If you paste XML from another source, switch to raw XML mode first, paste your content, then switch back to form mode to verify it was parsed correctly.

Saving

Click Save to write changes back to the plist file on disk. Save is disabled if the XML contains validation errors.

Note: Only User Agent plist files (~/Library/LaunchAgents/) can be edited. Global agents and daemons are read-only.
